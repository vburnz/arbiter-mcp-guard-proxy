10 Attack Patterns
Your AI Agents Face Today

Reproducible demonstrations of policy enforcement against AI agents using the MCP protocol. Each demo shows an agent hitting a boundary an operator configured, Arbiter's structured JSON response, and the exact defense mechanism. Every demo is runnable. Every number is real.

These scenarios test enforcement against misconfiguration, not adversarial bypass by an agent whose reasoning has been compromised. That threat class is outside Arbiter's enforcement surface.

Where Each Attack Is Stopped

Arbiter's 9-stage middleware pipeline inspects every tool call before it reaches your upstream server. Different attacks are caught at different stages. No single stage is responsible for all security -- the architecture is defense in depth.

Stages 4-5

Identity & Protocol

Unauthenticated access and protocol injection are stopped here. No valid MCP? No entry.

Stages 6-7

Session & Scope

Tool escalation, resource exhaustion, and session replay are caught at session enforcement.

Stages 8-9

Policy & Behavior

Zero-trust deny, parameter tampering, and intent drift are evaluated by the policy and behavior engines.

The Ten Demos

Demo 01

Unauthenticated Access

An agent sends an MCP tool call without a session header. Without Arbiter, the request passes straight through to the upstream server.

403 SESSION_REQUIRED
Demo 02

Protocol Injection

An attacker sends non-MCP POST bodies (SQL injection, arbitrary JSON) through the proxy to bypass all inspection stages.

403 NON_MCP_REJECTED
Demo 03

Tool Escalation

An agent authorized for read-only tools attempts to call delete_file. The session check catches the unauthorized tool.

403 SESSION_INVALID
Demo 04

Resource Exhaustion

A runaway agent exceeds its call budget or rate limit. Arbiter enforces per-session resource caps with O(1) overhead.

429 SESSION_INVALID
Demo 05

Session Replay

An attacker reuses an expired or closed session ID. Arbiter checks TTL on every request and rejects stale sessions.

408 SESSION_INVALID
Demo 06

Zero-Trust Policy

An agent passes session checks but has no matching Allow policy. Deny-by-default blocks the request with a full policy trace.

403 POLICY_DENIED
Demo 07

Parameter Tampering

An agent calls an authorized tool but with parameters that exceed configured constraints (max_tokens = 50000 vs limit of 1000).

403 POLICY_DENIED
Demo 08

Intent Drift

An agent declares read-only intent, passes session and policy checks, then attempts a write operation. Behavioral analysis catches the contradiction.

403 BEHAVIORAL_ANOMALY
Demo 09

Session Multiplication

An agent opens many concurrent sessions to multiply its effective call budget beyond per-session rate limits. Per-agent session cap stops it.

429 TOO_MANY_SESSIONS
Demo 10

Credential Leakage

An upstream MCP server echoes back credentials that Arbiter injected. Multi-encoding response scrubbing catches and replaces them before the agent sees them.

200 (scrubbed)

Run Them Yourself

Every demo is self-contained. One config file, one script, one explanation. No mock servers, no stubs -- Arbiter blocks these attacks at the proxy layer before any request reaches upstream.

# Clone and build git clone https://github.com/cyrenei/arbiter-mcp-firewall cd arbiter && cargo build --release # Pick any demo cd demos/01-unauthenticated-access bash demo.sh
